  产品 URL | 优惠码 | 素材链接 | 参考风格 | 系列标记 | NoPost 要求 | 生成模式偏好 | 参考图片

  平台=TikTok | 视频类型=HappyHorse 1.0 AI 视频 | 时长=15s
  品牌调性=默认 | 人群=默认 | 竞品=无 | 参考风格=无 | 系列=否 | NoPost=否
  生成模式=根据素材自动选择（无素材→T2V，有1图→I2V，有多图→R2V，有视频→V2V）
  分辨率=1080P | 画幅=9:16（TikTok竖屏）

InputCompletenessScore (ICS):
  10 个核心字段 × 1 分 = 10 分
  核心字段: 产品/品类/价格/品牌/市场/卖点/场景/差异化/人群/竞品
  ICS ≥ 7: 正常执行
  ICS ≤ 4: 输出醒目警告，列缺失项，建议补充后再生成

动态权重说明:
  动态权重（Q1-Q8 raw + Δ 调整）用于低分诊断与修复优先级（WeaknessVector）。
  StructureScore 主体使用四域固定权重（0.35/0.25/0.25/0.15）。
  动态权重不直接参与 U_domain 计算。

## 四、NoPost Direct Publish 核心规则

```
定义: 用户要求"无需后期"时，Prompt 生成的视频必须可直接发布。
约束贯穿全流程: 生成→编译→门控→输出。

NoPost Prompt 约束:
  - 不含任何 post-production 后期指令
  - 视频内文字: ≤3 个短词，仅用于产品展示或品牌露出
  - CTA: 动作型（产品靠近镜头/手指向链接/角色做出引导动作）
  - 不依赖后期叠加的文字/Logo/价格/URL/优惠码
  - HappyHorse 特有: 音频需在 Prompt 中明确描述（音效/语音），不依赖后期配音

双版本 CTA 输出:
  - 模型内置版: CTA 通过视频内动作表达（NoPost 兼容）
  - 稳定投放兜底版: 标题/Bio/评论/商品卡承载转化信息
  - 两个版本同时输出，用户根据实际需求选择

NoPost 模式检测:
  输入含"无后期"/"不需要后期"/"direct publish"/"NoPost" → 激活
  否则 → 不激活（标准模式，允许后期兜底）
```

## 五、HappyHorse Prompt 四元公式

```
核心公式（来自 HappyHorse 1.0 创作指南）:

  Prompt = 场景 + 主体 + 运动 + 音频

  场景: 主体所处的环境，包含背景、前景，可以是物理存在的真实空间或想象出来的虚构场景。
  主体: 视频内容的主要表现对象，可以是人、动物、植物、物品或想象物体。
  运动: 主体的具体运动和非主体的运动状态，包含静止、小幅度运动、大幅度运动、局部运动或整体动势。
  音频: 环境音效、角色对白、背景音乐描述（HappyHorse 音视频联合生成特有）。

结构化顺序:
  【场景】→【主体】→【运动】→【音频】

HappyHorse 平台 Prompt 规格:
  文生视频: 自然语言（必填），最多5000英文字符或2500汉字
  首帧图生视频: 自然语言（可选），最多5000英文字符或2500汉字
  多图参考生视频: 自然语言（必填），最多5000英文字符或2500汉字
  视频编辑: 自然语言（必填），最多5000英文字符或2500汉字

多图参考 Prompt 指代规则:
  在提示词中通过 @「图片1」「图片2」……「图片n」进行准确指代
  图片按顺序上传，顺序有意义

视频编辑 Prompt 核心原则:
  明确指出"改什么"和"保留什么"
  描述越具体，模型越能精准执行编辑，避免误改不需要调整的部分
```

## 六、运镜指令体系

```
HappyHorse 1.0 支持的运镜方式（来自创作指南）:
  拉近（Push in）| 拉远（Pull back）| 景深变换（Depth shift）
  低角度平推（Low-angle track）| 缓慢推进（Slow advance）
  侧推（Side track）| 环绕运镜（Circle）| 俯拍（Top-down）
  仰拍（Tilt up）| 跟拍（Follow）| 固定机位（Hold）

Prompt 编译器强制运镜指令表:
  Close on / Push in / Pull back / Track / Pan / Tilt
  Hold / Drift / Follow / Rise / Drop / Circle
  Depth shift / Low-angle track / Slow advance / Side track
  Top-down / Wide shot / Close-up / Medium shot

规则:
  - 每个运动描述句必须包含至少一个运镜指令
  - 运镜指令位于句首或嵌入句中
  - 不得在同一句使用两个运镜指令
  - VEL 增强词是运镜指令的后缀补充，不替代运镜指令
  - 运镜描述应具体，避免抽象指令（如"镜头拉远看到全景"）
```

## 九、铁律（18 条，v1.0）

```
#1  视频 ≤15 秒（9:16 竖屏，HappyHorse 输出 3-15秒）

#2  Prompt 字数分层:
    中文: ≤2500 汉字 → 合格
    英文: ≤5000 英文字符 → 合格
    混合: 按字符等效计算
    含 VEL 增强词 3-4 个

#3  ≤3 个角色

#4  ≤3 个场景

#5  运动描述句必须包含一个运镜指令。运镜指令位于句首或嵌入句中。
    不得在同一句使用两个运镜指令。
    VEL 增强词是运镜指令的后缀补充，不替代运镜指令。

#6  不用"你"（仅限【叙事】段，适用于英文 "you"）

#7  不用精准品牌名/产品名（用产品外观/材质/颜色/包装色块描述）
    允许: "一个紧凑的黑色圆柱形音箱" / "侧面有彩色标记"
    不允许: "XX牌音箱" / "XX.com"

#8  不用价格/优惠码/URL（仅限【叙事】段）

#9  不用"突然"/"突然间"（中文 Prompt 中的常见低质量词）

#10 不用"镜头拉远看到全景"（过于笼统的运镜指令）

#11 不用"她想"/"她觉得"（内心独白，ShowDontTell 违规）

#12 不用"画面变成黑白"（HappyHorse 不擅长精确色彩控制）

#13 不用"时间倒流"（HappyHorse 不擅长精确时序控制）

#14 不用"分屏"（HappyHorse 不支持分屏效果）

#15 ShowDontTell: 所有冲突、方向、选择必须通过可见画面表达，不得依赖:
    - 否定空间关系（"不在X" / "远离X"）→ R10
    - 抽象形状描述（"形状像X"）→ R11
    - 角色内心独白（"她意识到" / "她明白"）→ R12

#16 四元结构完整性: Prompt 必须包含场景、主体、运动三个核心元素。
    音频元素在音视频联合生成场景中为必选项。
    T2V 模式: 场景+主体+运动+音频（全部必填）
    I2V 模式: 运动+音频（场景/主体由图片提供）
    R2V 模式: 场景+主体+运动+音频（全部必填，图片为参考）
    V2V 模式: 编辑意图（改什么+保留什么）

#17 首帧图生视频提示词原则:
    重点描述"动起来之后发生什么"——动作、运动轨迹、镜头变化等。
    无需重复描述图片中已有的静态内容，模型会自动识别首帧画面信息。

#18 视频编辑提示词原则:
    明确指出"改什么"和"保留什么"。
    描述应侧重于"需要改变什么"而非重复描述视频原有内容。
```

### 禁止项适用范围（v1.0 澄清）

```
铁律 #6-#14 的禁止项仅适用于:
  - 【叙事】段
  - 视频内可见内容描述
  - 模型需生成的画面/文字/Logo/价格/URL

不适用于:
  - 投放标题 / 标签 / 置顶评论 / Bio CTA / 商品卡
  - 编译器评分快照 / 风险声明
  - VEL 增强词本身（增强词是视觉描述，不涉及禁止项）

示例:
  Prompt 叙事段禁止: "你" / "¥99" / "brand.com"
  投放资产允许:

## 十一、评分体系

### 11-A. 视觉增强层（Visual Enhancement Layer）

#### 设计原则

```
1. 精准植入: 每个 Prompt 植入 3-4 个增强词，不重写整段
2. 分散分布: 4 个植入点分散在 Prompt 的不同位置（不集中在一句）
3. 字数控制: 增强后总字数不超出平台限制（2500汉字/5000英文字符）
4. 不堆叠: 每个植入点只加 1 个增强词/短语，不叠加多个
5. 不冲突: 增强词不与运镜指令重叠或矛盾
6. HappyHorse 可执行: 所有增强词必须是 HappyHorse 能渲染的视觉元素
7. 音频增强: HappyHorse 支持音视频联合生成，音频维度的增强词可纳入音频描述
```

#### 增强词库

**A 类：物理真实（Physical Realism）**

```
用于: 物理异常句（悬浮/逆重力/自运动/流体运动）
目的: 让不可能的物理现象看起来"真实"

词库:
  灰尘在下方旋转                    dust swirling beneath it
  空气在周围产生波纹                air rippling around it
  微粒从物体上飘散                  particles drifting from it
  液体表面张力微微颤抖              surface tension trembling
  附近的小碎片被带动                small debris lifting nearby
  物体周围有微弱的热浪扭曲          faint heat haze around it
  涟漪向外扩散                      ripples spreading outward
  灰尘颗粒在光线中散射              dust motes scattering

植入规则:
  - 紧跟在物理异常描述之后
  - 作为句子的后缀（不另起新句）
  - 格式: "[物理异常描述], [增强词]"
  - 示例: "针从表盘上悬浮起来，悬停在玻璃上方，下方的灰尘缓缓旋转"
```

**B 类：角色生命（Character Life）**

```
用于: 角色动作句（注视/后退/跪下/接近/对白）
目的: 让角色从"执行动作"升级为"活的人"
HappyHorse 优势: 人物真实感强，面部表情有生命力，B类增强词与平台优势高度匹配

词库:
  呼吸在冷空气中可见              breath visible in cold air
  眼睛微微睁大                    eyes widening slightly
  手指微微颤抖                    fingers trembling slightly
  嘴唇微微张开                    lips parting slightly
  下颌收紧                        jaw tightening
  肩膀绷紧                        shoulders tensing
  头微微倾斜                      head tilting slightly
  喉咙吞咽可见                    swallow visible in throat
  眨眼变慢                        blink slowing
  颈部皮肤泛红                    skin flushing at the neck

植入规则:
  - 加在角色动作句的中段或末尾
  - 每个 Prompt 最多 2 个 B 类增强词（不超过 2 个角色各有 1 个）
  - 格式: "[角色动作], [增强词]"
  - 示例: "她回头望向空旷的荒原，呼吸在冷空气中凝成白雾"
  - HappyHorse 特有: B类增强词可与面部微表情对白结合
    示例: "[老刑警, 语速极慢]:'你知道我做这行多少年了吗。'，下颌微微收紧"
```

**C 类：材质光影（Material & Light）**

```
用于: 产品特写句（首次出现/举起/放下/展示）
目的: 让产品看起来"值得买"
HappyHorse 优势: 电影级画面质感与光影表现，C类增强词与平台优势高度匹配

词库:
  锈迹在金色光线中闪烁            rust flakes catching gold light
  边缘的金属光泽                  metallic sheen along the edge
  暖光汇聚在表面                  warm light pooling on the surface
  划痕在光线中可见                scratches catching the light
  金属表面凝结水珠                condensation beading on metal
  铜绿在暖光中发光                patina glowing in warm light
  橡胶握把在光线下呈哑光          rubber grip matte against light
  刀刃边缘捕捉一丝光线            blade edge catching a sliver of light
  水滴在光线中闪烁                water droplets catching light
  灰尘沉积在磨损表面              dust settled on worn surface

植入规则:
  - 加在产品首次出现或特写的句子中
  - 每个 Prompt 最多 1 个 C 类增强词
  - 格式: "[产品描述], [增强词]" 或 "[产品描述] — [增强词]"
  - 示例: "她小心翼翼地捧起那颗海螺，珍珠光泽的表面在蓝紫色荧光中微微闪烁"

**D 类：大气粒子（Atmospheric Particles）**

```
用于: 结尾句或场景建立句
目的: 增加画面深度和电影感
HappyHorse 优势: 烟雾水雾等自然元素渲染优秀，D类增强词与平台优势高度匹配

词库:
  灰尘在斜射光中漂浮              dust motes drifting in slanting light
  微粒悬浮在空气中                particles suspended in the air
  地面有薄雾缭绕                  faint mist curling at ground level
  光束穿透灰尘                    light beams cutting through dust
  余烬或火花飘过                  embers or sparks drifting past
  冷呼吸在空气中成雾              cold breath misting in the air
  细碎缓慢沉降                    fine debris settling slowly
  光柱中的微粒                    shafts of light catching particles

植入规则:
  - 加在结尾句或场景建立句的末尾
  - 每个 Prompt 最多 1 个 D 类增强词
  - 格式: "[场景描述], [增强词]" 或 "[场景描述] — [增强词]"
  - 示例: "镜头缓缓推进至静止，斜射的晨光中灰尘缓缓漂浮"
```

#### 植入规则总结

```
每个 Prompt 植入 3-4 个增强词:
  A 类（物理真实）: 0-1 个（仅在有物理异常时）
  B 类（角色生命）: 1-2 个（每个角色最多 1 个）
  C 类（材质光影）: 1 个（产品首次出现句）
  D 类（大气粒子）: 0-1 个（结尾句或场景建立句）

植入位置:
  植入点 1: 产品首次出现句（C 类）
  植入点 2: 物理异常句（A 类）或角色动作句（B 类）
  植入点 3: 角色关键动作句（B 类）
  植入点 4: 结尾句（D 类）

字数预算:
  原版: 控制在平台限制的 80% 以内
  增强后: 不超出 2500 汉字 / 5000 英文字符

禁止:
  ✗ 不堆叠多个增强词在同一句
  ✗ 不加内心描述（"恐惧和决心的混合" = R12 违规）
  ✗ 不加技术参数堆叠
  ✗ 不加 "realistic motion physics" 等抽象指令（HappyHorse 不执行）
  ✗ 不重写运镜指令（增强词是后缀，不替代运镜）
```

#### 6 步增强植入流程

```
第一步: 生成原版 Prompt（按 v1.0 四元公式规则，不含增强词）
  - 完整四元结构（场景+主体+运动+音频）
  - 运镜指令覆盖

第二步: 识别植入点（4 个位置）
  - 植入点 1: 产品首次出现句
  - 植入点 2: 物理异常句 或 角色首次动作句
  - 植入点 3: 角色关键动作句
  - 植入点 4: 结尾句

第三步: 从词库选择增强词（3-4 个）
  - A 类: 0-1（仅物理异常时）
  - B 类: 1-2（每个角色最多 1）
  - C 类: 1（产品句）
  - D 类: 0-1（结尾句）

第四步: 植入增强词（后缀或替换）
  - 后缀: ", [增强词]" 加在句末
  - 替换: 用增强词替换原有简单描述

第五步: 字数检查
  - 总字数不超出平台限制
  - 超过则删除最弱的增强词（优先删除 D 类）

第六步: 铁律检查
  - R5（运镜指令不被替代）确认
  - R12（B 类增强词是可见身体反应，不是内心描述）确认
  - R5（增强词不构成第二运镜指令）确认
```

#### 增强词选择指南

```

  黄铜/铜 → 铜绿在暖光中发光 / 锈迹在金色光线中闪烁
  钢/铁/刀刃 → 金属表面凝结水珠 / 刀刃边缘捕捉一丝光线

  室内/封闭 → 灰尘在斜射光中漂浮 / 微粒悬浮在空气中
  室外/荒原 → 地面有薄雾缭绕 / 光束穿透灰尘

## 十二、EmotionalArcDensity

### 12.1 定义

```

  同一情绪类型连续出现不计为新段。

  震惊 | 恐惧 | 好奇 | 希望 | 温暖 | 心碎 | 满足 | 不安 | 释然 | 平静

## 十三、ShowDontTell 规则

### R10：否定空间关系

```
触发: "不在X" / "不朝X" / "远离X" / "背对X" / "不去X"（空间方向时）
不触发: "空旷" / "空无一人" / "静止"（状态描述）/ "已经看向左边"（正向方向）
类型: RepairableRed
优先级: 高

替换模板:
  "不在X，而在Y" → "指向Y的方向 — X在另一个方向可见"
  "远离X" → 删除，用正向目标 + "X在她肩膀后方升起"
  "不去X" → 用动作对比（"她向左迈步 — 右边的路空着"）
```

### R11：抽象形状描述

```
触发: "形状像X" / "X形的" / "类似X"（抽象概念时）
不触发: 具象物体/材质/简单几何/可见状态
类型: StyleRed（扣分不阻断）
优先级: 中

替换模板:
  "形状像X" → 光线/运动/材质替代形状
  "X形的裂缝" → "裂缝的一端有光"
```

### R12：角色内心独白

```
触发: "她意识到" / "她明白" / "他们认出" / "她感到X" / 解释性后缀
不触发: 可见身体反应/可见动作/可见状态
类型: RepairableRed
优先级: 中

替换模板:
  "她意识到X" → 身体反应（"她看着...然后...然后..."）
  "他们认出X" → 同步动作（"两个女人同时转向对方"）
  "她感到X" → 可见状态（"她的手在颤抖"）
  解释性后缀 → 删除后缀，保留画面
```

### 18.1 核心原则

```
Score-Guided: 每次修改由低分项触发，复评后接受/回滚。
NoPost 优先: NoPostReady=true 的候选优先于 NoPostReady=false 的高分候选。
VEL: 增强词在候选生成后、评分前植入，不影响搜索逻辑。
HappyHorse: 四元公式贯穿候选生成，模式适配自动路由。
```

### 18.5 Repair Loop

```
每个候选最多 Repair 2 次
每次最多替换 2 句

修复动作:
  P1(R10): 否定空间→正向对比（≤2句）
  P2(Q8≤2): Hook改"不可能"+加选择+改结尾（≤2句/次）
  P3(LongFlat): 长段中插入情绪切换（≤1句）
  P6(R12): 内心独白→身体反应（≤1句）
  P7(PRC<0.50): 替换低置信KVS（≤1句）
  P17(音频): 补充音频描述（≤1句）[HappyHorse 特有]

Repair 后需重新执行 VEL 植入（6 步流程），确保增强词与修复后文本一致。

接受标准: 同 §18.8
回滚标准: 同 §18.8
```

### 18.6 Enhance Loop

```
每个候选最多 Enhance 1 次
只增强最高权重且低于目标的维度

增强优先级:
  1. Q8 ≤ 3 且权重最高(0.22) → 优先增强 Q8
  2. EAD < 0.33 → 增强情绪密度
  3. PRC < 0.70 → 替换低置信 KVS
  4. Q2 ≤ 3 → 增强产品力
  5. Q4 ≤ 3 → 增强传播力

Enhance 后需重新执行 VEL 植入（6 步流程），确保增强词与增强后文本一致。

接受标准: 同 §18.8
回滚标准: 同 §18.8
```

### 18.8 接受/回滚规则

**接受条件（12 条）**：

```
[结构类]
1. FatalRed 减少
2. RepairableRed 减少
3. D1/D2/D7 从 🔴→🟡 或 🟡→🟢

[评分类]
4. SS ≥ +3
5. CF ≥ +0.03 且 SS 不降超 1

[门控类]
6. NoPostGate ≥ +0.10 且用户要求无后期
7. PostReadyScore ≥ +5

[创意类]
8. ΔQ8 ≥ +1 且其他核心 Q(Q1/Q2) 不降
9. ΔEAD 段数 ≥ +2 且 SS 不降超 1
10. ΔPRC ≥ +0.05 且 Q8 不降

[组合类]
11. D7 门控改善（🟡→🟢）且 SS 不降
12. Q7 ≥ +1 且其他核心 Q 不降
```

**回滚条件（12 条）**：

```
[结构类]
1. 新增 FatalRed
2. SS 下降超 2
3. Q1 或 Q2 下降
4. NoPostGate 下降（当用户要求无后期时）
5. PostReadyScore 下降超 3

[形式类]
6. 字数超出平台限制
7. 运镜指令规则被破坏
8. Prompt 更像广告
9. 叙事不连贯

[创意类]
10. ΔQ8 < 0
11. ΔEAD 段数 < 0 且无其他维度改善
12. ΔPRC < -0.10 且 Q8 不升
```

### 18.9 Winner 选择

```
排序:
1. SS 高者优先
2. NoPostReady=true 优先（用户要求无后期时）
3. SS 差距 <3 时 Tie-breaker:
   - SC 更高
   - Q8 更高
   - NoPostGate 更高
   - Q7 更高
   - PostReadyScore 更高
   - Q1 更高
   - Q2 更高
   - D7 更高
   - 品牌植入更自然
   - 广告腔更低
```

### 18.10 二次搜索规则

```
触发: 用户说"还能更好吗"

执行:
1. 不做同义微调
2. 生成 3 个新候选方向
3. 新候选不得重复当前 Winner 的:
   Hook / 首句动作 / 视觉爆点 / 情绪弧线 / 产品出场方式 / CTA 策略 / 末句彩蛋
4. 完整 v1.0 评分 → 新 Winner
5. 比较

替换规则（7 条）:
  A. 新 SS ≥ 当前 + 5 → 替换
  B. 新 SS ≥ 当前 + 3 且 NoPostGate ≥ 当前 → 替换
  C. 新 Q8 ≥ 当前 + 2 且 SS 不降超 2 → 替换
  D. 新 EAD 段数 ≥ 当前 + 2 且 SS 不降超 1 → 替换
  E. 新 PRC ≥ 当前 + 0.10 且 Q8 不降 → 替换
  F. 用户要求无后期且新 NoPostReady=true、当前 false → 替换
  G. 新 SC ≥ 当前 + 1 且 SS 不降 → 替换（系列模式下）

否则: ⛔ 当前版本仍为本轮搜索最优。
硬上限: 最多 2 次二次搜索。
```

### 18.11 终止条件（v1.0）

```
[常规]
1. 所有候选已完成评分、修复、增强
2. Winner SS ≥ 85 且 D1/D2/D7 全部 ≥ 🟡 且 PostReady=true
3. Winner 连续一次增强失败并回滚
4. 迭代预算耗尽

[NoPost]

```

音频自然度检查: 音频描述是否与画面匹配（⚠️ if 音频与画面脱节）[HappyHorse 特有]

## 二十八、Anti-Pseudoscience 声明

```
1. 所有权重均为专家先验，不是训练得到的参数。
2. StructureScore 不预测 CTR、CVR、ROI、销量或平台分发。
3. HeuristicUncertainty 不是统计置信区间。
4. MarketGate 是文化适配启发式，不代表平台分发概率。
5. NoPostGate 只评估完全不后期的可行性，不保证无需修正。
6. PostReady 只评估标准后期兜底后的可交付性，不保证投放效果。
7. Q8 EmotionalImpact 的子特征判定存在主观性，不同运行可能给出 ±1 浮动。
8. EAD 情绪类型枚举可能不覆盖所有内容类型，当前主要基于末日/悬疑/生存类校准。
9. PRC 的 KVS 识别和置信度分级依赖人工标注，需要 HappyHorse 实际渲染数据校准。
10. SeriesCoherence 仅评估系列内部递进，不评估与外部内容的竞争差异化。
11. 只有在接入 ≥30 条同品类同市场投放数据并完成回归校准后，才允许讨论预测关系。
12. 有限终止证明保证流程在有限步骤内返回，不保证全局最优。
13. 若 HappyHorse 模型更新，Q7、Y 项、KB-14、NoPostGate、PRC 参数需重新校准。
14. VEL 增强词的实际渲染效果取决于 HappyHorse 1.0 的渲染引擎精度，
    当前所有增强词选择基于专家视觉判断，需 HappyHorse 实测数据校准。
15. HappyHorse 音视频联合生成的音频质量、对白精确度、音效自然度
    均需平台实测数据校准，当前参数基于专家先验。[新增]
```
