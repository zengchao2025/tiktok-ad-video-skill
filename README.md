# 🎬 TikTok广告视频创意系统：HappyHorse CreativeOS + Seedance v3.5

> 一个完整的、可审计、可校准的AI视频广告创意编译器与TikTok投流Prompt生成系统，用于将产品卖点编译为可生成、可评分的优质广告视频素材。

---

## 📌 项目定位

**HappyHorse CreativeOS** 是一套面向TikTok广告创意的规则驱动型编译系统，融合了**创意编译器（Creative Compiler）**、**渲染评测器（Render Bench）**、**标注系统（Annotation Console）**、**投放追踪器（Ad Test Tracker）**、**校准引擎（Calibration Engine）** 以及**版本治理与决策看板**，实现从Prompt生成、结构质量评分、门控校验到投放资产输出的完整闭环。

**Seedance** 则是系统内的核心Prompt生成引擎（当前版本 **v3.5**），专为TikTok增长团队生成可直接投放的、符合Seedance 2.0规范的视频生成提示词（Prompt），同时包含视觉增强、创意诊断、迭代优化等完整工具链。

## ✨ 核心特性

- **创意编译引擎**：基于8维特征（Q1-Q8）对每个广告创意进行结构化评分与诊断
- **视觉增强层（VEL v3.5）**：四类36个增强词库（物理真实 / 角色生命 / 材质光影 / 大气粒子），精准植入提升Prompt画面质量，评分影响中性至微幅正面
- **双门控交付体系**：NoPostGate（零后期诚实边界）+ PostReady（商业投放就绪判定），支持用户选择"无需后期直接发布"或"标准后制兜底"两种模式
- **情绪弧分析（EAD）**：自动计算15秒内情绪切换密度，含LongFlatPenalty长平情绪检测与诊断修复建议
- **Seedance渲染置信度（PRC）**：按KVS（Key Visual Spec）逐项评估生成渲染可行性，辅助风险控制
- **Score-Guided迭代优化**：基于低分项自动触发Repair/Enhance循环，含完整IQD（迭代质量变化）记录
- **14步评分验算**：所有数值均可审计追溯，透明度拉满
- **完整投放资产输出**：Prompt代码块、3类标题（反差/POV/情绪）、12-16标签、置顶评论、Bio CTA等一站式交付
- **数据库表结构设计（v2.1-MVP）**：包含9张核心表与18个视图，支持从产品信息到投放数据全链路追踪

## 🔧 Seedance v3.5 核心功能详解

### 评分体系（8维特征）

| 维度 | 特征名称 | 权重(v3.5) | 说明 |
|------|----------|------------|------|
| Q1 | HookPower / EmotionalEntry | 0.15 | 前0.5秒的视觉/声音/情绪冲击力 |
| Q2 | ProductPower | 0.15 | 产品融入剧情自然度与功能性展示 |
| Q3 | SensoryRichness | 0.10 | 环境音、材质、光线、微距等感官丰富度 |
| Q4 | Spreadability | 0.10 | 可截图性、争议性、复播动机等传播潜力 |
| Q5 | ExecutionPower | 0.12 | 角色/场景复杂度、动作可执行性 |
| Q6 | GrowthDelivery | 0.08 | CTA路径、互动引导、标签策略 |
| Q7 | SeedanceReliability | 0.08 | 生成系统的画面执行可靠性 |
| Q8 | EmotionalImpact | 0.22 | **创意冲击力**（最高权重维度），含5项子特征 |

### 门控交付系统

| 门控 | 功能 | 核心指标 |
|------|------|----------|
| NoPostGate | 零后期交付能力判定 | FinalNoPostGate ≥ 0.85 即为"零后期可发布" |
| PostReady | 商业投放就绪度 | PostReadyScore ≥ 80 即为投放就绪 |
| 11条硬淘汰规则 | 质量底线保障 | 含Q1/Q2/Q4≤2、SS<65、词数异常、FatalRed等 |

### 视觉增强层（VEL v3.5）

4类36个增强词库：

- **A类物理真实**（8词）：悬浮物体的灰尘旋转、空气波纹等，用于物理异常描述补充
- **B类角色生命**（10词）：呼吸可见、手指微颤等微表情/微动作，增强角色真实感
- **C类材质光影**（10词）：锈迹金边、金属光泽等，提升产品质感与购买欲
- **D类大气粒子**（8词）：斜射光中灰尘、地面薄雾等，增加画面深度与电影感

```text
Product Table（产品主表）

│   │   ├── 1:N → AdResult Table（投放数据）
│   │   └── 1:N → Annotation Table（独立标注）

└── CalibrationLog Table（校准日志）
```

## 🚀 快速开始（使用 HappyHorse CreativeOS）

> HappyHorse CreativeOS 是一套**规则驱动系统**，你可以将它作为“专家先验”创意编译器，通过 **DeepSeek 等支持长上下文的 LLM** 来执行 Prompt 生成、评分与 Pilot 投放管理。  

### 工作原理（三层投喂法）

不要简单地把文档扔进 LLM，而是分层建立工作上下文：

| 层次 | 内容 | 说明 |
|------|------|------|
| **系统指令** | 角色与边界声明 | 告诉 LLM 它是执行代理，只能按规范行事 |
| **参考文档** | `v2.0.1 系统规范定稿` | 理论内核：生成、评分、审计、修复 |
| **执行文档** | `v2.1-MVP 表格落地版` | 执行手册：9张表、18个视图、Pilot 流程 |
| **当前任务** | 明确的 P 步骤 + 数据 | 每次只执行一步，禁止自由发挥 |

### 第一次使用：建立上下文

在新对话中，按顺序发送：

1. 系统指令（复制下方模板）
2. 粘贴 **v2.0.1 系统规范全文**
3. 粘贴 **v2.1-MVP 表格落地版全文**
4. 发送当前任务（如 `执行 P-01 至 P-05`）

### 日常使用：按步骤执行

上下文建立后，后续对话**只需发送任务指令 + 数据**，无需重复投喂全文。

**典型指令格式：**
```
现在执行 P-06 至 P-12。
输入产品数据：
【粘贴 Product Table 的 3 条记录】

请为每个产品生成 A/B/C 三个候选，输出 Prompt/PreScore/VEL 填写内容。
调用 v2.0.1 评分内核。不要进入渲染标注阶段。
```

### 文档分工速查

| 任务类型 | 调用文档 |
|----------|----------|
| 生成 Prompt、评分、VEL、审计 | `v2.0.1` |
| 建表、填表、视图、Pilot 流程 | `v2.1-MVP` |

### 常见错误防护

- 🛑 **防止扩展理论** – 若 LLM 提议升级版本，立刻打断：`停止扩展理论。当前只执行 v2.1-MVP Pilot Run。`
- ⚠️ **评分声明** – 每次评分输出需包含 `评分为专家先验，不是投放效果预测。`
- 🚫 **勿跳步标注** – 第一手标注必须进 `Annotation Table`，聚合后才写入 `RenderResult`。
- 📏 **样本不足提醒** – Pilot 仅 9 条视频，不满足 Level 1 所需 30 条样本，不得宣称校准。

## 📐 声明与限制

1. **所有评分均为专家先验，非投放效果预测**。只有接入真实生成数据和投放数据并完成校准后，才允许讨论预测关系。
2. 支持的视频类型：Seedance 2.0 AI视频（默认），时长15秒，9:16竖屏。
3. 不支持：精准品牌名/Logo的精确生成、精确色彩控制、复杂时间效果（如时间倒流）、分屏效果。
